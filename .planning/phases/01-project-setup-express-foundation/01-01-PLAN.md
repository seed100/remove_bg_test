## Objective
Initialize the TypeScript Express project, implement layer-based folder structure, and setup centralized error handling.

## Tasks

```xml
<task id="01-01-01" autonomous="true">
  <objective>Initialize npm project, install dependencies, and configure TypeScript</objective>

    - package.json
  </read_first>
  <action>
    Initialize npm project if not exists (`npm init -y`).
    Install runtime dependencies: `express`, `cors`, `dotenv`.
    Install dev dependencies: `typescript`, `@types/node`, `@types/express`, `ts-node-dev`, `jest`, `ts-jest`, `@types/jest`, `supertest`, `@types/supertest`.
    Create `tsconfig.json` with `target: ES2022`, `module: CommonJS`, `outDir: ./dist`, `rootDir: ./src`, `strict: true`.

  </action>
  <acceptance_criteria>

  </acceptance_criteria>

<task id="01-01-02" autonomous="true">
  <objective>Setup Express application, routing skeleton, and centralized error handler</objective>
  <read_first>
    - src/app.ts
  </read_first>
  <action>
    Create `src/middlewares/errorHandler.ts` that exports an Express error handling middleware (`(err, req, res, next)`). It should return `{ error: err.message || "Internal Server Error" }` with status `err.statusCode || 500`.
    Create `src/app.ts` that initializes `express()`, uses `express.json()`, `cors()`.
    Add a health check route `GET /api/health` that returns `{ status: "ok" }`.
    Add a 404 catch-all middleware before the error handler that passes an error "Not Found" with status 404 to `next()`.
    Use the `errorHandler` middleware at the end of `src/app.ts`.
    Create `src/server.ts` that imports `app` from `./app` and listens on `process.env.PORT || 3000`.
  </action>
  <acceptance_criteria>
    - `src/middlewares/errorHandler.ts` contains `(err: any, req: Request, res: Response, next: NextFunction)`
    - `src/app.ts` contains `app.get('/api/health'`
    - `src/app.ts` contains `app.use(errorHandler)`
    - `src/server.ts` contains `app.listen`
  </acceptance_criteria>
</task>
```
